1 Введение

1.1              Область применения

Модернизированная АИС РСМ в части государственной кадастровой оценки объектов недвижимости города Москвы предназначена для автоматизации выполнения государственной кадастровой оценки и использование результатов оценки для целей, установленных законодательством РФ.

 

1.2              Краткое описание возможностей

Модернизированная АИС РСМ в части государственной кадастровой оценки объектов недвижимости города Москвы предназначена для решения следующих задач:

­   определение кадастровой стоимости учтённых в ЕГРН объектов недвижимости;

­   определение кадастровой стоимости вновь учтённых объектов недвижимости, ранее учтённых объектов недвижимости в случае внесения в ЕГРН сведений о них и объектов недвижимости, в отношении которых произошло изменение их количественных и (или) качественных характеристик, в период между датой проведения последней государственной кадастровой оценки и датой проведения очередной государственной кадастровой оценки;

­   мониторинг рынка недвижимости.

 

1.3              Уровень подготовки персонала

Персонал АИС РСМ разделён на категории:

-          обслуживающий персонал;

-          пользователи.

 

Уровень подготовки пользователей должен обеспечивать:

-        базовые навыки работы на персональном компьютере с современными операционными системами (клавиатура, мышь, управление окнами и приложениями, файловая система);

-        базовые навыки использования интернет-браузера (настройка типовых конфигураций, установка подключений, доступ к веб-сайтам, навигация, формы и другие типовые интерактивные элементы веб-интерфейса);

-        знание основ информационной безопасности.

 

К обслуживающему персоналу дополнительно предъявляются следующие требования:

-        знать принципы построения систем управления базами данных;

-        иметь навыки работы с серверным и телекоммуникационным оборудованием;

-        иметь расширенные знания в области поддержки пользователей;

-        быть обученными основам администрирования операционных систем, серверов приложений и серверов баз данных.

2 Назначение и условия применения

2.1              Виды деятельности, цели

Модернизированная АИС РСМ в части кадастровой оценки объектов недвижимости города Москвы предназначена для автоматизации деятельности по определению кадастровой стоимости учтенных в ЕГРН объектов недвижимости, построения расчетной модели с использованием методов статического анализа.

Объектами автоматизации является совокупность технологических процессов:

­   ведение данных об объектах недвижимости, подлежащих государственной кадастровой оценке, и их характеристиках;

­   учет решений суда;

­   учет деклараций;

­   учет решений комиссии по оспариванию кадастровой стоимости;

­   нормализация и гармонизация данных об объектах недвижимости города Москва, полученных из различных источников;

­   выполнение настройки параметров расчета объектов недвижимости города Москва;

­   построение математической модели расчета кадастровой стоимости объектов недвижимости города Москва.

4 Описание операций

4.1         Учет решений суда

Посредством подсистемы государственной кадастровой оценки реализована возможность загрузки и ведения данных о судебных мерах, предпринятых в части переоценки кадастровой стоимости объектов. Для удобства объекты разделены на разделы (см. Рисунок 5 – Судебные решения).

 

Рисунок 5 – Судебные решения

 

Существует 2 метода внесения данных в блок «Судебные решения»:

·         Загрузка файла в формате Excel;

·         Ручное внесение данных.

 

4.1.1        Загрузка данных

Загрузка данных осуществляется с помощью функции загрузки данных, доступной из боковой панели функций (см. Рисунок 6 – Загрузка данных)

Рисунок 6 – Загрузка данных

 

При использовании функции, осуществляется открытие карточки загрузки данных с возможностью перехода выбора файла на диске.

 

4.1.2        Ручное внесение данных

Предусмотрена возможность внесения данных по судебным решениям методом ручного заполнения карточки объекта. Для внесения нового объекта используется функция «Добавить» (см. Рисунок 7 – Добавить).

Кроме того, реализована возможность автоматического сопряжения сведений о характеристиках объекта недвижимости из основной базы данных при вводе информации о кадастровом номере объекта недвижимости и дате определения его кадастровой стоимости.

Рисунок 7 – Добавить

 

4.1.2.1   Заполнение карточки объекта

Карточка объекта состоит из следующих блоков:

·         Основные данные (см. Рисунок 8 – Основные данные объекта);

·         Расчет ДРС (см. Рисунок 9 – Расчет ДРС).

Рисунок 8 – Основные данные объекта

 

Рисунок 9 – Расчет ДРС

 

Блок «Основные данные» содержит в себе следующие, в том числе обязательные к заполнению поля (обязательные к заполнению поля помечены):

·         Кадастровый номер объекта недвижимости, в отношении кадастровой стоимости которого инициирован судебный спор (обязательное к заполнению поле);

Подсистема «Объекты недвижимости» предназначена для хранения и накопления данных по объектам недвижимости, подлежащим кадастровой оценке.

При переходе в подсистему «Объекты недвижимости» пользователю становятся доступны представление, а также механизмы «Загрузка данных по списку» (1), «Выгрузка данных по списку» (2), просмотр карточки «Единицы оценки» (3) (см. Рисунок 46 – Объекты недвижимости).

4.4.1        Загрузка данных по списку

При использовании функции «Загрузка данных по списку» осуществляется переход на карточку загрузки (см. Рисунок 47 – Загрузка данных по списку).

 

Рисунок 47 – Загрузка данных по списку

 

Загрузка осуществляется из файлов в формате MS Excel. Для загрузки данных необходимо выполнить следующую последовательность действий:

1.      Нажать на кнопку – «Загрузить». При этом откроется окно выбора файла на диске;

2.      Соотношение данных из загружаемого файла с данными хранящимися в подсистеме. После выбора файла с данными, наименования колонок файла будет отображено в блоке «Колонки в файле Excel» (2). В блоке «Атрибуты реестра» (4) перечислены атрибуты, имеющиеся в системе. Для сопоставления данных из загружаемого файла с данными атрибутов, необходимо выделить соответствующие значения в обоих блоках, после чего нажать кнопку «Сопоставить» (3);

3.      Сопоставленные значения будут отображены в блоке «Сопоставление» (5). После сопоставление всех необходимых полей существует возможность выбора наиболее приоритетных данных проставлением галочки – флажка напротив приоритетного сопоставления. Так же при больших объемах загрузки желательно активировать режим фоновой загрузки. Для активации режима фоновой загрузки необходимо активировать флажок «Фоновая Загрузка»;

4.      Нажать кнопку «Загрузить».

 

4.4.2        Единица оценки

Карточка единицы оценки делится на следующие информативные блоки:

·         Единица оценки. В блоке представлены значения основных факторов идентифицирующих объект кадастровой оценки;

·         Результат оценки. В блоке представлена текущая кадастровая стоимость, основные параметры, повлиявшие на расчет, а также документ, в рамках которого расчет производился;

·         Данные стоимости ЕГРН. В блоке представлены данные полученные в рамках загрузки XML ЕГРН;

·         Факторы стоимости. В блоки перечисляются стоимостные факторы, влияющие на расчет кадастровой стоимости объекта.

В рамках расчетной подсистемы производится настройка тура, построение модели, группировка, загрузка документов на расчет кадастровой стоимости.

При переходе в раздел пользователь попадает на рабочий стол расчетной подсистемы (см. Рисунок 48 – Рабочий стол расчетной подсистемы)

Для начала работы в расчётной подсистеме в первую очередь необходимо произвести создание и настройку тура. Для перехода к созданию и настройке тура необходимо перейти в подраздел «Туры», создать и настроить тур, группировку и метод расчета для каждой подгруппы, пользуясь функциональностью раздела (см. Следующим этапом работы в расчетной подсистеме является настройка группировки объектов недвижимости. Данная операция производится с помощью загрузки файла группировки.

По завершении выполнения всех работ по созданию и подготовке тура к расчету, необходимо перейти к построению математической модели на основе библиотеки «WolframAlpha». Образцовый коэффициент качества модели стремится к значению, максимально приближенному к 1, и является параметром контроля, чем выше значения коэффициента качества, тем точнее будет выполнен расчет кадастровой стоимости.

После настройки всех справочников необходимо перейти к формированию заданий на оценку. Для формирования задания на оценку необходимо нажать на кнопку «Создать задание на оценку», после чего заполнить все необходимые данные в открывшейся карточке «Создать задания на оценку», а также приложить XML-файл, после чего нажать «Сохранить» (см. Рисунок 50 – Создание задания на оценку). На основании ранее заполненных справочных данных могут быть созданы задания на оценку различных типов (годовые, ежедневные, пересчет исходного перечня). Выбор типа задания осуществляется пользователем из выпадающего списка при создании. После завершения процесса загрузки Система уведомит пользователя о завершении загрузки. Данные о результатах оценки можно просмотреть в разделе «Задания на оценку» (см. Рисунок 51 – Задание на оценку).

Карточка «Задания на оценку» содержит в себе:

1.      Основные данные (основные данные выбранного задания на оценку).

2.      Единицы оценки (список объектов недвижимости, по которым производится перерасчет кадастровой стоимости в рамках выбранного задания на оценку).

3.      Журнал загрузки (данные о дате и продолжительности загрузки XML-файлов).

4.5.1        Интеграция с программным обеспечением статистического анализа

Интеграция с программным обеспечением статического анализа осуществляется без участия пользователя, на основе библиотек «WolframAlpha» и используется для построения математических моделей на основании распределённых на сегменты рынка объектов-аналогов (сделок и предложений). Библиотеки «WolframAlpha» позволяют выполнять вычисления и получать подробный результат с возможностью быстрого переключения между различными вариантами моделей. «WolframAlpha» является общедоступной системой для математического анализа и может применяться, в том числе, и в задачах математического моделирования любой сложности.

5 Аварийные ситуации

При сбое в работе аппаратуры восстановление нормальной работы системы должно производиться после перезагрузки операционной системы.

При ошибках в работе аппаратных средств (кроме носителей данных и программ) восстановление функции системы возлагается на ОС.

При ошибках, связанных с программным обеспечением (ОС и драйверы устройств), восстановление работоспособности возлагается на ОС.

При неверных действиях пользователей, неверных форматах или недопустимых значениях входных данных, система выдает пользователю соответствующие сообщения, после чего возвращается в рабочее состояние, предшествовавшее неверной (недопустимой) команде или некорректному вводу данных. В данной ситуации пользователю необходимо обратиться к администратору системы с информацией о сообщении, выданном системой, и условиях, при которых данное сообщение было получено.
