1.2. Уровень подготовки администратора

Для установки и настройки Системы администратору необходимо обладать следующими знаниями и умениями:

-        принципы построения систем управления базами данных;

-        навыки работы с серверным и телекоммуникационным оборудованием;

-        расширенные знания в области поддержки пользователей;

-        навыки общего администрирования ОС, серверов приложений и серверов баз данных;

-        Инсталляция и настройка СУБД PostgreSQL.

1.3. Комплект поставки для инсталляции

Комплект поставки осуществляется на носителе информации и представляет собой набор файлов и каталогов:

-           папка OPS\Script\ – скрипты БД

-           папка OPS\OPSPublishedWebsites\ – веб-приложение для развертывания на IIS

-           Папка OPS\WindowsService\ – служба для выполнения фоновых процессов (импорты, экспорты и др.)

 

2. Инструкция по развертыванию системы

2.1. Общие положения

Для развертывания Системы должны быть выполнены следующие работы:

-        обновление схемы БД;

-        развертывание сервера приложений.

2.2. Установка на сервер БД

Установка на сервер БД включает в себя выполнение скриптов БД на заранее установленной и настроенной схеме БД.

В случае необходимости установить СУБД PostgreSQL (если она не установлена) согласно документации в соответствии с типом операционной системы, которая будет использоваться для установки (https://www.postgresql.org/files/documentation/pdf/11/postgresql-11-A4.pdf).

2.2.1. Скрипты БД

Для установки скриптов платформы необходимо:

-        Открыть папку OPS\Script\;

-        Открыть файл ExecuteSqlFiles.config и исправить строку подключения к базе данных в XML-узле connectionStrings;

-        Запустить файл EcecuteSqlFiles.exe.

2.4. Запуск службы фоновых процессов

Установка службы фоновых процессов включает в себя следующие шаги:

-        открыть основную директорию службы C:\CIPJSWindowsService, представленную в соответствии с (Рисунок 7);

Рисунок 7 – Основная директория службы фоновых процессов

-        скопировать новые файлы с диска с дистрибутивом Системы: OPS\WindowsService;

-        открыть скопированный файл Core.WindowsService.dll.config. При необходимости отредактировать данный файл, исправив в нем данные соединения с БД в блоке connectionStrings и другие настройки в соответствии с (Рисунок 8);

-        В дальнейшем вызов файла «_start Core.WindowsService.bat» нужно добавить в автозапуск Windows.

3. Настройка прав доступа

Для работы в Системе в соответствии с разграничениями прав доступа необходимо создать и настроить роли в системе, а также создать и включить пользователей в требуемые роли.

3.1. Создание и настройка ролей

Для создания новой роли в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Список ролей;

-        Нажать кнопку «Создать роль» (Рисунок 9);

-        В открывшемся окне добавить Наименование роли и указать список функций, которые необходимо включить в роли;

-        После сохранения созданная роль становится доступной в общем списке ролей.

Рисунок 9 – Добавление новой роли

 

Для настройки существующей роли в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Список ролей;

-        Выбрать требуемую роль (Рисунок 9);

-        Указать список функций, которые необходимо включить/исключить в роли;

-        Сохранить внесенные изменения.

 

3.2. Создание и настройка пользователей

Для создания нового пользователя в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Список пользователей;

-        Нажать кнопку Операции/Добавить пользователя (Рисунок 10);

-        В открывшемся окне добавить: логин пользователя, имя пользователя для отображения в документах, пароль. А также выбрать роли, в которые необходимо включить пользователя.

-        После сохранения пользователь становится доступным в общем списке пользователей системы.

Рисунок 10 – Добавление нового пользователя

 

Для изменения пароля у пользователя в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Список пользователей;

-        выбрать требуемого пользователя;

-        Нажать кнопку Операции/Изменить пароль (Рисунок 10);

-        В открывшемся окне ввести новый пароль, а также его подтверждение;

-        сохранить внесенные изменения.

 

Для изменения роли у пользователя в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Список пользователей;

-        выбрать требуемого пользователя;

-        выбрать новые роли для пользователя;

-        сохранить внесенные изменения.

3.3. Создание и подразделений

Для создания нового подразделения в Системе необходимо:

-        запустить Систему с правами Администратора;

-        зайти в раздел Панель управления – Учетные записи пользователей – Подразделения;

-        Нажать кнопку Операции/Добавить подразделение;

-        В открывшемся окне добавить: код, наименование, руководителя.

-        После сохранения подразделение становится доступным в общем списке подразделений системы.

 

Подразделения необходимы только для объединения пользователей, принадлежность к подразделению не влияет на права доступа к функциям системы.

5. Настройка справочников

Для настройки справочных значений необходимо:

 

-        Перейти в раздел «Панель управления – Настройка справочников»;

-        Выбрать необходимый справочник и убедиться, что в колонке «Редактирование открыто» стоит значение «Да»;

6. Журналы мониторинга состояния системы

Система ведет следующие журналы мониторинга системы:

-        Журнал ошибок – данный журнал необходим для оперативного исправления кода системы, в случае возникновения ошибок (исключений), не обрабатываемых текущим кодом;

-        Диагностика – данный журнал необходим для анализа операций, которые выполняются слишком долго. В данном журнале фиксируются операции, которые выполнялись более 10 секунд.

 

Для доступа к данным журналам необходимо перейти в раздел:

-        «Панель управления – Контроль работы системы – Журнал ошибок» (Рисунок 15);

Рисунок 15 – Журнал ошибок

7. Фоновые процессы

Фоновые процессы запускаются службой фоновых процессов по расписанию или по запросу. Для просмотра типов фоновых процессов, которые есть в системе необходимо перейти в раздел «Панель управления – Фоновые процессы - Типы» (Рисунок 17).

 

 

Рисунок 17 – Типы фоновых процессов

 

На закладке «Процессы» отображается журнал фактического запуска процесса выбранного типа. Та же информация без фильтра по типу доступна в разделе «Панель управления – Фоновые процессы – Процессы».

В разделе «Панель управления – Фоновые процессы – Службы» отображается журнал запуска служб фоновых процессов. Как правило служба запускается один раз при загрузке сервера приложений, при запуске служба добавляет запись в данный журнал.

8. Журналы загрузки

В разделе «Панель управления – Журналы загрузки» доступны журналы импорта данных из БТИ и ЕГРН, а также журнал формирования объектов страхования на основе импортированных данных.

9. Настройка печатных форм

Для настройки печатных форм, используемых в системе необходимо зайти в раздел «Панель управления – печатные формы». В данном разделе отображаются печатные формы, существующие в Системе (Рисунок 18).

Рисунок 18 – Список печатных форм

 

Для добавления новой печатной формы необходимо нажать кнопку «Добавить» и ввести: код, внутреннее имя и заголовок.

Для редактирования шаблона печатной формы необходимо нажать на кнопку «Редактировать» (Рисунок 19).

Рисунок 19 – Редактирование печатной формы

Создание и редактирование печатных форм выполняется с использованием инструмента FastReport Online Designer в соответствии с руководством пользователя на данный продукт (https://www.fastreport.ru/ru/product/fast-report-online-designer/documentation/).
